# AI Contribution Policy Template

This project accepts AI-assisted contributions when the contributor remains accountable for the work.

## Contributor Requirements

- Understand the change you submit.
- Link the issue or explain the problem before opening a large PR.
- Keep PRs small and focused.
- Disclose meaningful AI assistance in the PR body when requested by the project.
- Run tests and explain any missing test coverage.
- Do not paste maintainer feedback into an AI tool and resubmit changes you do not understand.

## PR Disclosure Example

```markdown
AI assistance: Used Codex to draft tests after I wrote the implementation. I reviewed and edited the generated code before submitting.
```

## Maintainer Review

Maintainers may ask for more context, split a PR, require tests, or close low-context contributions without deep review.
